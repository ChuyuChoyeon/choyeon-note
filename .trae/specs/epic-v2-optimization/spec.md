# Choyeon Note v2.0 史诗级优化 - Product Requirement Document

## Overview
- **Summary**: Choyeon Note 史诗级深度优化，参考 Obsidian 关系图、VSCode 编辑器体验，全面增强拼写检查、代码高亮、图表支持、自动更新、Bing 壁纸背景等核心功能，打造生产级可落地的 Markdown 笔记软件。
- **Purpose**: 解决当前编辑器功能薄弱、知识图谱简陋、拼写检查不完整、无自动更新、缺乏代码高亮和图表支持等核心问题，使产品达到 Obsidian/VSCode 级别的专业体验。
- **Target Users**: 知识工作者、程序员、学生、研究人员等需要高质量 Markdown 笔记工具的用户。

## Goals
- 知识图谱达到 Obsidian 级别的可视化效果和交互体验
- 拼写检查支持忽略字典、选中弹出工具栏、阅读模式隐藏
- 编辑器支持代码高亮、Mermaid 图表、各类 Markdown 扩展语法
- 完整的自动更新机制（检测/下载/安装/更新日志）
- 设置页面功能完善（字体大小、代码高亮主题、反馈、Bing 壁纸）
- 整体性能优化，无明显 bug 和逻辑问题
- 构建打包并发布到 GitHub Release

## Non-Goals (Out of Scope)
- 多人协作功能
- 云端同步功能
- 移动端适配
- 插件系统
- 数据库存储引擎替换

## Background & Context
项目基于 Vue 3 + Vite + Tailwind CSS + Electron 构建，已实现基础的笔记管理、编辑器、知识图谱等功能。但与 Obsidian、Typora、VSCode 等成熟产品相比，在编辑器体验、可视化效果、功能完整性上还有较大差距。本次优化旨在大幅提升产品核心竞争力。

## Functional Requirements

### FR-1: 知识图谱深度优化（参考 Obsidian）
- 力导向布局算法优化，节点分布更自然
- 节点大小反映笔记重要性（字数/链接数/标签权重）
- 连线粗细反映关联强度
- 悬浮高亮：鼠标悬停节点时，高亮其邻居节点，其他节点变暗
- 拖拽节点后位置记忆
- 缩放和平移支持（滚轮缩放、拖拽平移）
- 点击节点跳转到对应笔记
- 节点颜色按标签/文件夹区分
- 局部图谱模式：显示当前笔记的直接关联
- 搜索过滤：输入关键词过滤显示节点

### FR-2: 拼写检查增强
- 自定义忽略字典（用户可添加单词到本地字典）
- 选中错误单词弹出浮动工具栏（忽略一次/全部忽略/添加到字典）
- 工具栏与选中区域不冲突，鼠标可平滑移动到工具栏
- 编辑模式和实时模式都支持拼写检查操作
- 阅读模式不显示拼写错误下划线
- 字典持久化到 localStorage
- 设置页面可查看和管理自定义字典

### FR-3: 编辑器深度优化
- 代码块语法高亮（支持 20+ 常用语言）
- 代码高亮主题可切换（设置页面选择，实时预览）
- Mermaid 图表支持（流程图、时序图、甘特图、饼图等）
- 数学公式支持（KaTeX）
- 表格增强（对齐、语法高亮）
- 工具栏添加图表插入按钮
- 实时预览模式下图表正常渲染
- 阅读模式下图表和代码高亮正常显示
- Task List 任务列表复选框交互
- 脚注支持

### FR-4: 自动更新功能
- 启动时自动检测更新
- 设置页面手动检查更新按钮
- 更新内容（Release Notes）支持 HTML 渲染
- 下载进度显示
- 下载完成后提示重启安装
- Electron 自动更新集成（electron-updater）
- 更新渠道设置（稳定版/测试版）

### FR-5: 设置页面增强
- 字体大小调节功能修复，实时生效
- 代码高亮主题选择下拉框，带实时预览
- 反馈按钮，点击跳转到 GitHub Issues 提议页面
- Bing 每日壁纸背景功能（开关 + 预览）
- 自定义字典管理界面
- 更新设置（自动检查、检查渠道）
- 重置所有设置按钮

### FR-6: Bing 每日壁纸背景
- 调用 Bing 每日壁纸 API 获取图片
- 背景图片应用到主内容区域
- 支持磨砂玻璃/透明度调节
- 每日自动更新
- 可手动刷新当前壁纸
- 设置中可开关

## Non-Functional Requirements

### NFR-1: 性能
- 知识图谱 100 个节点以内 60fps
- 编辑器 10000 字以内输入无延迟
- 页面切换动画 60fps
- 启动时间 < 3 秒

### NFR-2: 可靠性
- 自动更新失败不影响现有功能
- 拼写检查不阻塞主线程
- 图表渲染失败有降级显示

### NFR-3: 可维护性
- 代码模块化，新增图表类型只需注册插件
- 主题系统统一，新增主题只需配置 CSS 变量
- 完善的错误边界处理

### NFR-4: 安全性
- 自动更新签名验证
- Bing 壁纸请求跨域安全处理
- 用户字典数据本地存储，不上传

## Constraints
- **Technical**: Vue 3 + Vite + Electron + Tailwind CSS，保持现有技术栈
- **Business**: 单机应用，无后端服务
- **Dependencies**: 
  - highlight.js / Prism.js 用于代码高亮
  - mermaid 用于图表
  - katex 用于数学公式
  - electron-updater 用于自动更新

## Assumptions
- 用户设备可访问 GitHub 用于自动更新
- Bing 壁纸 API 可公开访问
- 拼写检查基于英语词典，中文拼写检查暂不支持
- 自动更新仅支持 Windows 平台

## Acceptance Criteria

### AC-1: 知识图谱优化
- **Given**: 用户有 20+ 篇笔记，包含多个标签和关联
- **When**: 用户打开知识图谱视图
- **Then**: 节点自然分布，大小和颜色有区分度，悬浮高亮正常，拖拽和缩放流畅
- **Verification**: `human-judgment`

### AC-2: 拼写检查增强
- **Given**: 编辑器中有拼写错误的单词
- **When**: 用户选中错误单词
- **Then**: 弹出浮动工具栏，可选择忽略或添加到字典，阅读模式无错误标记
- **Verification**: `human-judgment`

### AC-3: 代码高亮和图表
- **Given**: 笔记中包含代码块和 mermaid 图表
- **When**: 用户切换到实时/阅读模式
- **Then**: 代码有语法高亮，图表正常渲染，设置中可切换高亮主题且有预览
- **Verification**: `human-judgment`

### AC-4: 自动更新
- **Given**: 有新版本发布
- **When**: 应用启动或用户手动检查
- **Then**: 检测到更新，显示更新日志（HTML 渲染），可下载并重启安装
- **Verification**: `human-judgment`

### AC-5: 设置页面增强
- **Given**: 用户在设置页面
- **When**: 调整字体大小、切换代码主题、开启 Bing 壁纸
- **Then**: 所有设置实时生效，字体大小正常，背景显示 Bing 壁纸
- **Verification**: `human-judgment`

### AC-6: 构建和发布
- **Given**: 所有功能开发完成
- **When**: 执行构建和打包
- **Then**: 构建成功，安装包生成，发布到 GitHub Release
- **Verification**: `programmatic`
